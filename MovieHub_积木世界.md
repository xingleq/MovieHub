# MovieHub 积木世界 UI 设计规范
版本：1.0
适用项目：MovieHub（Flutter / Dart，Windows 优先，兼容 Android TV）
设计目标：将儿童影视桌面应用统一为“积木玩具世界”视觉体系，兼顾儿童友好、电视端可读性、跨分辨率适配与后续 AI 协同开发。

---

## 1. 产品定位

MovieHub 是一款面向儿童与家庭使用的本地影视媒体中心，核心体验包括：

- 本地影视库浏览
- TMDB 刮削与海报墙展示
- 动画、电影、电视剧分类
- 收藏与播放记录
- 抽卡奖励
- 家长控制与休息提醒
- Windows 桌面端
- 后续兼容 Android TV / 大屏电视端

本规范只约束视觉与交互，不改变当前业务逻辑。

---

## 2. 视觉主题定义

### 2.1 主题名称

**MovieHub 积木世界**

### 2.2 主题关键词

- 积木
- 玩具
- 方块世界
- 乐高式结构感
- 我的世界式体素感
- 明亮
- 冒险
- 儿童友好
- 清晰易读
- 可点击感强
- 电视端友好

### 2.3 视觉原则

1. 所有 UI 元素都应具有“可搭建、可拼装、可按压”的感觉。
2. 形状以方形、圆角矩形、积木凸点、块状按钮为主。
3. 页面结构清晰，不为了可爱牺牲可读性。
4. 装饰元素不能遮挡海报、文字和主要操作。
5. 儿童模式强调趣味，家长设置页强调清晰和可信。
6. 保持统一，不同页面不能各自使用完全不同的视觉语言。
7. 尽量避免使用整张位图作为功能 UI；优先使用 Flutter 组件、SVG 图标和可复用装饰资源。

## 3. 品牌视觉语言

### 3.1 Logo 建议

MovieHub Logo 建议使用积木拼字效果：

- “Movie” 使用暖橙色或黄色积木字
- “Hub” 使用蓝色积木字
- Logo 外轮廓有 2–4 px 深色描边
- 可附带一颗黄色积木星星
- 可搭配绿色积木小恐龙作为品牌吉祥物

Logo 不建议用于所有页面大面积展示，只在：

- 启动页
- 首页左上
- 关于页面
- 空状态页面
- 家长设置中心顶部

### 3.2 吉祥物

统一使用一个原创的绿色积木小恐龙，避免每个页面出现不同风格角色。

吉祥物状态建议：

- 默认站立
- 开心挥手
- 观看影片
- 抱着爆米花
- 睡觉
- 提醒休息
- 找不到内容
- 设置成功
- 抽卡惊喜
- 家长锁定

吉祥物只作为情绪反馈和页面装饰，不承担核心信息传达。

## 4. 色彩规范

### 4.1 主色

| 名称 | 色值 | 用途 |
|---|---|---|
| 冒险蓝 | `#2D78FF` | 主按钮、选中状态、焦点边框 |
| 积木黄 | `#FFC629` | 首页高亮、奖励、播放主操作 |
| 草地绿 | `#2DBE60` | 动画分类、成功状态、儿童内容 |
| 珊瑚红 | `#FF5A4F` | 关闭、警告、限制状态 |
| 魔法紫 | `#8454E8` | 电影、抽卡、稀有内容 |
| 天空青 | `#28BFD6` | 辅助按钮、刷新、轻提示 |

### 4.2 中性色

| 名称 | 色值 | 用途 |
|---|---|---|
| 云朵白 | `#FFFFFF` | 卡片背景、导航背景 |
| 天空浅蓝 | `#EFF7FF` | 页面底色 |
| 积木灰 | `#6B7280` | 次级文字 |
| 深色文字 | `#1E2A3A` | 标题、正文 |
| 软边框 | `#D8E8F8` | 卡片边框 |
| 禁用灰 | `#C8CFD8` | 禁用状态 |

### 4.3 色彩使用规则

- 单个页面的主色不超过 3 种。
- 页面背景保持浅色，主操作使用高饱和色。
- 红色只用于错误、关闭、限制和危险操作。
- 黄色主要用于播放、奖励和选中首页。
- 蓝色用于系统级导航、焦点和主要交互。
- 不使用大面积纯黑。
- 影视背景图上必须叠加深色渐变，确保文字清晰。

## 5. 字体与文字层级

### 5.1 字体原则

- 中文字体应圆润但清晰。
- 导航和正文不使用过度装饰的卡通字体。
- 页面大标题可以适度使用更厚重的圆体。
- 文字必须保持为真实文本，不要做成图片。
- 电视端优先考虑 3 米距离可读性。

### 5.2 推荐字体

优先顺序：

1. 思源黑体 / Source Han Sans CN
2. 阿里巴巴普惠体
3. HarmonyOS Sans SC
4. 微软雅黑
5. 系统默认无衬线字体

### 5.3 字号建议

#### Windows 1920×1080

| 层级 | 字号 | 字重 |
|---|---:|---|
| 首页主标题 | 48–64 | 700–800 |
| 页面标题 | 32–40 | 700 |
| 模块标题 | 24–30 | 700 |
| 卡片标题 | 18–22 | 600 |
| 正文 | 16–18 | 400–500 |
| 辅助文字 | 14–16 | 400 |
| 小标签 | 12–14 | 600 |

#### Android TV / 大屏

整体建议放大 15%–30%。

### 5.4 文案风格

儿童界面文案：

- 简短
- 积极
- 易懂
- 避免技术术语

示例：

- “继续冒险”
- “开始播放”
- “今天休息一下吧”
- “明天再来抽卡”
- “这里还没有影片”
- “请让爸爸妈妈帮助设置”

家长页面可使用正常技术文案。

## 6. 形状、圆角与描边

### 6.1 圆角

| 元素 | 圆角 |
|---|---:|
| 小标签 | 8 px |
| 普通按钮 | 10–12 px |
| 导航项 | 12–16 px |
| 卡片 | 16–20 px |
| 大型 Hero 面板 | 20–28 px |
| 弹窗 | 24–32 px |

积木风不代表完全直角。建议使用“方中带圆”的结构。

### 6.2 描边

- 默认描边：1 px `#D8E8F8`
- 可点击卡片：1.5–2 px
- Focus 状态：3 px 冒险蓝
- 高价值奖励卡：可使用 2 px 金色描边
- 深色背景上的白色卡片，可增加 1 px 半透明白描边

### 6.3 阴影

积木 UI 应有轻微立体感，但不能厚重。

推荐：

```text
普通卡片：
0 6 18 rgba(30, 80, 140, 0.10)

悬停 / Focus：
0 10 28 rgba(45, 120, 255, 0.24)

主按钮：
0 5 0 深色同系底边
```

按钮可使用“底部硬阴影”模拟积木按键。

## 7. 间距系统

统一使用 8 px 基础网格。

建议间距：

- 4 px：图标内部微调
- 8 px：紧凑元素
- 12 px：图标与文字
- 16 px：按钮内部
- 24 px：卡片内部
- 32 px：模块之间
- 48 px：页面大区域
- 64 px：Hero 与内容区
- 80–96 px：电视端大区块

所有页面必须遵守统一间距，不允许随机使用 13、17、29 等无规律数值。

## 8. 图标规范

### 8.1 图标风格

图标统一采用：

- 积木块感
- 像素 / 体素轮廓
- 两段或三段配色
- 深色描边
- 轻微高光
- 形状简洁
- 小尺寸仍可识别

### 8.2 图标内容

基础图标应包括：

- 搜索
- 首页
- 动画
- 电影
- 电视剧
- 抽卡
- 收藏
- 设置
- 播放
- 暂停
- 返回
- 关闭
- 最小化
- 最大化
- 文件夹
- 媒体库
- 刮削
- 刷新
- 播放记录
- 家长控制
- 外观
- 休息
- 喝水
- 护眼
- 奖励
- 星星
- 筛选
- 排序
- 网格
- 列表
- 用户
- 锁

### 8.3 SVG 使用要求

- 默认使用 SVG。
- ViewBox 必须统一。
- 不在 SVG 中写死页面背景色。
- 图标应允许通过 Flutter 修改主色。
- 不使用复杂滤镜、外链图片和大量渐变。
- 推荐尺寸：16 / 20 / 24 / 32 / 48 / 64。
- 导航图标一般使用 20–24 px。
- 电视端可放大至 28–32 px。
- 保留 2 px 安全边距。
- 线性图标与填充图标不得混用。

### 8.4 图标状态

每个导航图标至少支持：

1. 默认
2. Hover
3. Focus
4. Selected
5. Disabled

默认不建议切换不同图片，优先通过颜色、背景、描边和缩放控制。

## 9. 顶部导航栏规范

### 9.1 结构

```text
[搜索] [首页] [动画] [电影] [电视剧] [抽卡] [收藏] [设置]
```

右侧独立放置：

```text
[最小化] [最大化] [关闭]
```

### 9.2 外观

- 导航整体为白色积木长条
- 顶部可有少量凸点装饰
- 高度建议 56–72 px
- 圆角 16–20 px
- 轻描边与轻阴影
- 导航项之间可用细分隔线或积木拼接缝

### 9.3 导航项状态

#### 默认

- 白色或浅灰底
- 深灰文字
- 彩色小图标

#### Hover / Focus

- 轻微上浮或放大至 1.04
- 外描边为冒险蓝
- 背景变浅蓝
- 图标颜色更鲜明

#### Selected

- 彩色积木按钮
- 白色或深色高对比文字
- 底部硬阴影
- 当前页不得只靠颜色识别，应同时改变背景和形状

### 9.4 电视端

- 必须支持方向键切换
- Focus 状态必须明显
- 禁止只依赖鼠标 Hover
- 返回键应恢复到上一级焦点
- 导航项之间切换时使用 120–180 ms 动画

## 10. 页面布局总则

### 10.1 Windows 安全区域

以 1920×1080 为基准：

- 左右安全边距：32–48 px
- 顶部安全边距：16–24 px
- 内容区域尽量控制在 1680×880 左右
- 窗口按钮不得压住导航和主内容

### 10.2 电视端安全区域

以 1280×720 为基准：

- 左右安全边距：80–100 px
- 上下安全边距：48–72 px
- 核心文字与操作必须在安全区内
- 装饰背景允许超出安全区

### 10.3 响应式规则

禁止通过拉伸整张图片适配页面。

应使用：

- `LayoutBuilder`
- `MediaQuery`
- `Expanded`
- `Flexible`
- `AspectRatio`
- `FittedBox`
- `Wrap`
- 响应式断点

建议断点：

```text
小窗口：< 1280
标准桌面：1280–1919
大桌面：>= 1920
电视模式：根据平台或遥控模式判定
```

## 11. 首页规范

### 11.1 页面目标

首页负责：

- 快速播放
- 继续观看
- 内容推荐
- 展现品牌视觉
- 提供儿童最容易理解的入口

### 11.2 Hero 区

结构建议：

```text
[深色渐变影视背景]

左侧：
影片标题
一句简介
主按钮：立即播放
次按钮：查看详情
轮播指示器

下方：
继续观看 / 热播 / 推荐卡片
```

### 11.3 Hero 视觉

- 背景可以是影片剧照，但不应强行体素化。
- UI 装饰使用积木风，不破坏影视内容本身。
- 底部增加深色渐变，保证卡片和文字可读。
- 主标题可使用积木描边效果。
- 主播放按钮使用黄色或蓝色积木按钮。

### 11.4 内容卡片

首页海报卡片建议：

- 海报本身保持原图
- 外层使用彩色积木框
- 顶部可增加“热播”“推荐”“继续观看”小标签
- 底部展示标题、进度、收藏星星
- Focus 时上浮和放大

不建议把所有影视海报都改成积木画风，否则会失去内容辨识度。

## 12. 动画、电影、电视剧列表页

### 12.1 页面结构

```text
顶部导航
页面标题
搜索栏
筛选 / 排序 / 网格切换
内容海报网格
```

### 12.2 视觉原则

- 页面背景使用天空浅蓝。
- 搜索栏做成白色长积木块。
- 筛选按钮使用彩色小积木按钮。
- 海报区保持高信息密度。
- 不使用大面积 Hero，避免影响查找效率。
- 可选增加一条小型推荐横幅，不超过页面高度 25%。

### 12.3 海报网格

Windows 1920 宽度建议：

- 6–8 列
- 卡片间距 20–28 px

电视端建议：

- 5–6 列
- 卡片尺寸更大
- 每行切换焦点时自动滚动

## 13. 详情页规范

### 13.1 页面结构

```text
背景剧照 + 渐变遮罩
海报
标题
年份 / 类型 / 时长 / 评分
简介
播放按钮
收藏按钮
剧集列表
相关推荐
```

### 13.2 积木化方式

- 信息标签使用积木小块
- 播放按钮使用黄色大按钮
- 收藏用黄色星星积木
- 剧集编号使用可按压方块
- 不把电影剧照整体改成像素或积木风

## 14. 收藏页规范

收藏页重点是整理，不使用首页 Hero。

结构：

```text
我的收藏
全部 / 动画 / 电影 / 电视剧
最近收藏 / 名称 / 年份
海报网格
```

空状态：

- 吉祥物抱着空盒子
- 文案：“这里还没有收藏”
- 按钮：“去看看影片”

## 15. 抽卡页规范

### 15.1 页面定位

抽卡页允许比其它页面更梦幻、更鲜艳，但仍保持积木世界主题。

### 15.2 页面结构

```text
今日抽卡 Hero 区
中央神秘卡牌
抽卡状态
抽卡按钮
抽卡小助手
我的卡包
筛选标签
卡牌网格
```

### 15.3 视觉

- 背景可使用积木星空、积木云朵和发光平台。
- 卡牌边框保留现有稀有度颜色。
- 抽卡按钮必须是页面最强操作。
- SSR 等高稀有度可增加金色积木粒子。
- 卡包筛选标签使用小积木胶囊。
- 不使用过多闪烁动画，避免刺激儿童。

## 16. 设置页规范

### 16.1 形态

设置页保留独立页面，不做成完整弹窗。

推荐结构：

```text
左侧积木分类导航
右侧设置内容
顶部标题与统计
```

分类：

- 媒体库
- 刮削
- 播放
- 外观
- 家长控制
- 关于

### 16.2 设置卡片

- 白色大卡片
- 16–20 px 圆角
- 浅蓝边框
- 标题 + 说明 + 控件
- 每张卡只承载一个主题
- 避免大量小统计块平铺
- 重要状态可用图标积木块展示

### 16.3 家长控制

使用绿色、蓝色为主，红色仅表示限制或危险操作。

家长密码、观看时长、休息时长和抽卡次数应独立分组。

## 17. 弹窗规范

### 17.1 通用结构

```text
半透明遮罩
白色积木弹窗
右上关闭按钮
吉祥物或状态图标
主标题
说明文字
状态信息
主按钮
可选次按钮
```

### 17.2 尺寸

Windows：

- 小弹窗：420×280
- 标准弹窗：520×360
- 大弹窗：640×440

电视端：

- 宽度不超过安全区的 55%
- 字体放大
- 主按钮高度至少 64 px

### 17.3 关闭按钮

用户已明确要求休息类弹窗可以关闭：

- 右上角必须有明显关闭按钮
- 支持 `Esc`
- 电视端支持返回键
- 关闭不等于取消规则，仅关闭当前提示
- 若属于强制休息，关闭后页面仍应保持暂停状态，由业务逻辑决定

### 17.4 倒计时休息弹窗

必须包含：

- 标题：“休息时间到啦！”
- 倒计时
- 休息提示
- 关闭按钮
- 主按钮：“知道了，开始休息”

倒计时使用大号数字，不能只使用环形进度。

### 17.5 当日次数用完弹窗

必须包含：

- 标题：“今天的观看次数已用完”
- 今日已观看次数
- 每日可观看次数
- 明日提示
- 关闭按钮
- 主按钮：“知道了”

不使用强烈红色，避免制造惩罚感。

---

## 18. 按钮规范

### 18.1 主按钮

- 高度：48–56 px
- 电视端：60–72 px
- 水平内边距：24–32 px
- 圆角：10–14 px
- 有底部硬阴影
- 最多一个主按钮

示例：

- 立即播放
- 开始抽卡
- 保存设置
- 开始休息

### 18.2 次按钮

- 白底
- 彩色描边
- 文字与图标同色
- 不使用强阴影

### 18.3 幽灵按钮

- 无背景
- 仅文字和图标
- 用于“查看详情”“稍后处理”

### 18.4 危险按钮

- 红色
- 只用于删除、移除、重置、关闭程序
- 必须有确认提示

- Default
- Hover
- Focus
- Pressed
- Disabled
- Loading

Pressed 状态应向下移动 2–3 px，模拟积木按下。

## 20. Focus、Hover 与遥控器交互

### 20.1 所有可交互元素必须有 Focus

包括：

- 导航项
- 海报卡片
- 按钮
- 筛选项
- 设置项
- 剧集按钮
- 弹窗关闭按钮

### 20.2 Focus 样式

推荐组合：

- Scale：1.04–1.08
- 蓝色 3 px 边框
- 外发光
- 阴影增强
- 文字或图标亮度提高

### 20.3 动画时间

- Hover：120 ms
- Focus：150–180 ms
- 页面切换：220–300 ms
- 弹窗出现：180–240 ms
- 抽卡动画：可更长，但需提供跳过能力

### 20.4 电视端导航

- 使用 Flutter `FocusNode`
- 使用 `Shortcuts` 与 `Actions`
- 明确设置上下左右焦点关系
- 列表离屏时自动滚动
- 返回键优先关闭弹窗，再返回上级页面

## 27. 最终设计方向总结

## 28. 像素字体与特殊字体规范

### 28.1 使用目标

像素字体用于强化 MovieHub 的积木、方块、游戏和冒险氛围，但不得替代全部界面字体。

推荐使用比例：

```text
像素字体：10%–20%
常规无衬线字体：80%–90%
```

像素字体适合制造视觉记忆点，普通字体负责保证长时间阅读和电视端可读性。

### 28.2 适用场景

像素字体适合用于：

- MovieHub Logo
- 首页主标题
- “今日抽卡”等活动标题
- SSR / S / A / B / C 稀有度文字
- 倒计时数字
- “热门”“推荐”“继续观看”等短标签
- 关卡感、奖励感、冒险感较强的模块标题
- 空状态中的短句
- 少量强调数字

不建议用于：

- 影片长简介
- 设置页说明文字
- 搜索结果正文
- 家长控制说明
- 多行按钮文案
- 长段中文
- 法律、许可、版本等信息

### 28.3 推荐字体组合

#### 中文像素标题字体

可优先测试：

- Ark Pixel Font / 方舟像素字体
- Zpix / 最像素
- 其他明确支持简体中文、繁体中文和常用标点的开源点阵字体

#### 英文与数字像素字体

可优先测试：

- Press Start 2P
- DotGothic16
- Silkscreen
- VT323

#### 正文与设置页字体

继续使用：

- 思源黑体 / Source Han Sans CN
- HarmonyOS Sans SC
- 阿里巴巴普惠体
- 微软雅黑
- 系统默认无衬线字体

推荐组合：

```text
中文标题：Ark Pixel Font 或 Zpix
英文与数字：Press Start 2P
正文与设置：思源黑体或 HarmonyOS Sans SC
```

### 28.4 字体查找关键词

在 GitHub、Google Fonts 或字体网站搜索时，可使用：

```text
中文 像素字体
Chinese pixel font
CJK pixel font
bitmap font Chinese
pixel font TTF
pixel game font
retro font
arcade font
点阵字体
复古游戏字体
```

### 28.5 字体授权要求

下载字体后必须检查：

- License
- Commercial use
- Embedding
- Redistribution
- Modification
- 是否允许打包进桌面软件或 APK

优先选择：

- SIL Open Font License 1.1
- Apache License 2.0
- 其他明确允许软件嵌入和分发的开源许可证

谨慎使用：

- 仅个人使用
- 禁止商业使用
- 禁止再分发
- 许可描述模糊
- GPL 但未说明字体例外

字体许可证文件应与项目一起保存，例如：

```text
licenses/fonts/
├── ark-pixel-font-LICENSE.txt
├── press-start-2p-OFL.txt
└── source-han-sans-LICENSE.txt
```

不要向用户单独分发字体源文件；仅在许可证允许时随应用嵌入。

### 28.6 字体资源目录

```text
assets/
└── fonts/
    ├── pixel/
    │   ├── ArkPixel.ttf
    │   ├── Zpix.ttf
    │   └── PressStart2P-Regular.ttf
    │
    └── sans/
        └── SourceHanSansCN-Regular.otf
```

文件名应保持稳定，不要在代码中引用临时下载名称。

### 28.7 Flutter 配置参考

在 `pubspec.yaml` 中统一注册：

```yaml
flutter:
  fonts:
    - family: ArkPixel
      fonts:
        - asset: assets/fonts/pixel/ArkPixel.ttf

    - family: Zpix
      fonts:
        - asset: assets/fonts/pixel/Zpix.ttf

    - family: PressStart2P
      fonts:
        - asset: assets/fonts/pixel/PressStart2P-Regular.ttf

    - family: MovieHubSans
      fonts:
        - asset: assets/fonts/sans/SourceHanSansCN-Regular.otf
```

使用示例：

```dart
Text(
  '今日抽卡',
  style: const TextStyle(
    fontFamily: 'ArkPixel',
    fontSize: 32,
    height: 1.2,
    letterSpacing: 1,
  ),
)
```

中英文分开使用：

```dart
Text.rich(
  const TextSpan(
    children: [
      TextSpan(
        text: 'MovieHub ',
        style: TextStyle(fontFamily: 'PressStart2P'),
      ),
      TextSpan(
        text: '积木世界',
        style: TextStyle(fontFamily: 'ArkPixel'),
      ),
    ],
  ),
)
```

### 28.8 像素字体字号规则

像素字体对字号较敏感，应尽量使用清晰的整数阶梯。

推荐测试：

```text
12 / 16 / 20 / 24 / 32 / 40 / 48 / 64
```

如果字体基于固定点阵网格，可优先使用其原始字号的整数倍，例如：

```text
12 / 24 / 36 / 48
```

避免大量使用：

```text
17 / 23 / 29 / 37
```

这些字号容易导致笔画模糊、像素边缘不整齐或字符粗细不一致。

### 28.9 阴影与描边

像素字体适合硬边效果，不适合模糊发光。

推荐：

```dart
const TextStyle(
  fontFamily: 'ArkPixel',
  fontSize: 32,
  shadows: [
    Shadow(
      offset: Offset(2, 2),
      blurRadius: 0,
      color: Color(0xFF174A8B),
    ),
  ],
)
```

可使用：

- 1–3 px 硬边阴影
- 高对比描边
- 两层错位投影
- 积木色块背景

避免：

- 大范围高斯模糊
- 多层霓虹发光
- 细小字号叠加强描边
- 低对比度文字

### 28.10 中英文回退策略

像素英文字体通常不包含中文。不得依赖系统自动回退后仍保持统一视觉。

建议：

- 中文标题显式使用中文像素字体
- 英文和数字显式使用英文像素字体
- 正文显式使用常规中文字体
- 不在同一个 `Text` 中混合不可控的字体回退
- 混排时使用 `Text.rich` 分别指定字体

### 28.11 电视端可读性

电视端使用像素字体时：

- 字号至少比桌面端大 15%–30%
- 标题与背景对比度必须足够
- 不连续使用超过两行
- 不在高速滚动列表中大面积使用
- Focus 后不能因缩放导致文字像素边缘严重抖动
- 必须在 1280×720 和 1920×1080 下分别检查
